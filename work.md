# ТЗ ДЛЯ CURSOR: 2D ARENA SURVIVOR НА PHASER 3 + VUE 3

## 1. Цель проекта

Разработать браузерную 2D игру в жанре arena survivor / auto-shooter.

Игрок управляет героем на арене, уклоняется от волн врагов, герой автоматически атакует ближайших противников. За убийства выпадает опыт, игрок повышает уровень и выбирает улучшения. Волны усиливаются со временем, каждые несколько волн появляется босс.

Игра должна быть оптимизирована для браузера и публикации на Яндекс Играх.

---

## 2. Технологии

Использовать:

* Vue 3
* Vite
* Phaser 3
* TypeScript
* Pinia для состояния вне Phaser
* CSS для интерфейса
* LocalStorage для сохранений

Не использовать:

* тяжёлые 3D библиотеки;
* сложную физику Matter.js;
* генерацию всего проекта одним огромным файлом;
* хаотичную архитектуру без модулей.

## 4. Игровой цикл

Основной цикл:

1. Игрок появляется в центре арены.
2. Таймер запускает волну.
3. Враги спавнятся за пределами видимой области.
4. Враги идут к игроку.
5. Герой автоматически стреляет в ближайшего врага.
6. Убитые враги оставляют XP.
7. Игрок собирает XP.
8. При повышении уровня игра ставится на паузу.
9. Игрок выбирает 1 из 3 улучшений.
10. Игра продолжается.
11. Каждые 5 волн появляется босс.
12. После смерти игрока показывается экран результатов.

## 5. Управление

ПК:

* WASD — движение
* Стрелки — движение
* ESC — пауза

Мобильное управление:

* виртуальный джойстик слева
* автоатака работает без кнопки атаки

Игрок не целится вручную. Оружие стреляет автоматически.

## 6. Игрок

Параметры игрока:

```ts
player = {
  hp: 100,
  maxHp: 100,
  speed: 180,
  level: 1,
  xp: 0,
  xpToNextLevel: 11,
  damageMultiplier: 1,
  fireRateMultiplier: 1,
  projectileSpeedMultiplier: 1,
  pickupRadius: 80,
  armor: 0,
}
```

Игрок должен:

* двигаться плавно;
* не выходить из логики арены;
* получать урон при контакте с врагами;
* иметь короткую неуязвимость после получения урона;
* собирать XP рядом с собой;
* иметь простую анимацию idle/run.

## 7. Оружие

На старте у игрока одно базовое оружие:

### Energy Bolt

```ts
weapon = {
  id: "energy_bolt",
  name: "Energy Bolt",
  damage: 10,
  fireRate: 0.45,
  projectileSpeed: 450,
  projectileLifetime: 1400,
  pierce: 0,
  projectileCount: 1,
  spreadAngle: 0,
}
```

Логика:

* оружие ищет ближайшего врага;
* стреляет автоматически;
* пуля летит в позицию врага;
* если врагов нет — не стреляет;
* пули уничтожаются по таймеру или при попадании;
* использовать object pooling для пуль.

## 8. Враги

Минимум 5 типов врагов:

### 1. Crawler

* слабый
* быстрый
* мало HP

```ts
hp: 10

```

### 2. Tank

* медленный
* много HP

```ts
hp: 45

xp: 4
```

### 3. Runner

* очень быстрый
* мало HP

```ts
hp: 8

```

### 4. Shooter

* держит дистанцию
* стреляет по игроку

```ts
hp: 18

xp: 3
```

### 5. Splitter

* после смерти делится на 2 маленьких врага

```ts
hp: 25
speed: 55
damage: 10
xp: 3
```

## 9. Волны

Уровень не должен генерироваться процедурно сложной картой.

Нужно сделать:

* бесконечный тайловый фон;
* процедурный спавн врагов;
* постепенное усиление волн.

Волна длится 60 секунд.

Каждая волна имеет:

```ts
wave = {
  index: 1,
  duration: 60,
  enemyBudget: 100,
  spawnInterval: 1.2,
  enemyTypes: ["crawler"],
  boss: false
}
```

Каждая следующая волна:

* увеличивает enemyBudget;
* уменьшает spawnInterval;
* открывает новые типы врагов;
* усиливает HP и урон врагов.

Формула усиления:

```ts
enemyHp = baseHp * (1 + waveIndex * 0.12)
enemyDamage = baseDamage * (1 + waveIndex * 0.08)
enemySpeed = baseSpeed * (1 + waveIndex * 0.02)
```

Каждая 5-я волна:

* появляется босс;
* обычные враги продолжают спавниться, но медленнее;
* сверху HUD показывает BOSS.

---

## 10. Спавн врагов

Враги должны появляться за пределами экрана.

Алгоритм:

1. Получить позицию камеры.
2. Выбрать случайную сторону экрана.
3. Добавить отступ 80–160 пикселей за пределами экрана.
4. Создать врага.
5. Враг движется к игроку.

Важно:

* враги не должны появляться прямо на игроке;
* не создавать слишком много врагов за один кадр;
* использовать object pooling;
* ограничить максимум живых врагов.

```ts
maxAliveEnemies = 180
```

## 11. Боссы

Первый босс на 5-й волне.

Босс должен иметь:

```ts
boss = {
  hp: 1200,
  speed: 35,
  damage: 25,
  attackCooldown: 3,
  xpReward: 50
}
```

Поведение босса:

* медленно идёт к игроку;
* иногда делает рывок;
* иногда создаёт круговой залп;
* имеет большую полоску HP сверху;
* после смерти даёт много XP и сундук улучшения.

---

## 12. Улучшения

При повышении уровня показывать 3 случайные карточки.

Примеры улучшений:

### Damage Up

+15% урон.

### Fire Rate Up

+12% скорость атаки.

### Move Speed Up

+10% скорость движения.

### Projectile Speed Up

+15% скорость пуль.

### Max HP Up

+20 максимального HP.

### Heal

Восстановить 30 HP.

### Pierce Up

Пули пробивают +1 врага.

### Double Shot

+1 дополнительный снаряд.

### Magnet

+25% радиус сбора XP.

Улучшения должны храниться в `upgradesConfig.ts`.

---

## 13. XP и уровни

XP выпадает в виде маленьких сфер.

Логика:

* враг умирает;
* создаётся XP orb;
* если игрок рядом — XP orb летит к игроку;
* при касании добавляется XP;
* при заполнении XP bar вызывается Level Up.

Формула опыта:

```ts
xpToNextLevel = Math.floor(10 + level * 8 + level * level * 1.5)
```

## 16. Арена

На первом этапе не делать сложную процедурную карту.

Сделать:

* большой бесконечный фон из тайлов;
* декоративные объекты;
* псевдо-бесконечное поле вокруг игрока;
* камера следует за игроком.

Декор:

* камни;
* кристаллы;
* бочки;
* трещины;
* черепа;
* энергетические пятна.

Декор не должен мешать передвижению на первом этапе.

## 17. Производительность

Обязательно:

* object pooling для врагов;
* object pooling для пуль;
* object pooling для XP;
* не создавать новые объекты каждый кадр;
* не использовать тяжёлые эффекты;
* ограничить количество живых врагов;
* отключать обновление объектов далеко от камеры;
* использовать Phaser Arcade Physics;
* не использовать Matter Physics.

Цель:

* 60 FPS на ПК;
* стабильная работа в браузере;
* нормальная работа на средних мобильных устройствах.

## 22. Критерии готовности

Игра считается рабочей, если:

* игрок может двигаться по арене;
* враги появляются волнами;
* враги идут к игроку;
* герой автоматически стреляет;
* враги получают урон и умирают;
* выпадает XP;
* игрок повышает уровень;
* появляется выбор улучшения;
* волны усиливаются;
* каждая 5-я волна вызывает босса;
* HUD показывает актуальные данные;
* после смерти показывается Game Over;
* игра работает без критических лагов.
