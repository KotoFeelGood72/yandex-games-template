# GDD: «Кот Слияние»

**Жанр:** casual physics merge / drop merge game  
**Платформа:** Web, Яндекс Игры, мобильные браузеры, desktop  
**Технологии:** Vue 3, TypeScript, Matter.js, Canvas / PixiJS на следующем этапе  
**Цель документа:** описать полный геймдизайн игры, core loop, залипательность, прогрессию, монетизацию, рекламу и места для in-app purchases.

---

## 1. Краткое описание игры

«Кот Слияние» — залипательная физическая merge-игра, где игрок сбрасывает сверху милые объекты: молоко, корм, рыбок, игрушки, котят и редких котов. Одинаковые объекты сталкиваются, катятся, толкаются и сливаются в новый объект следующего уровня.

Главный смысл игры — не прохождение сложных уровней, а приятный бесконечный процесс: бросить объект, посмотреть на физику, получить слияние, открыть новый объект, набрать больше очков и продержаться дольше.

Игра должна вызывать ощущение:

- «ещё один бросок»;
- «сейчас получится большое слияние»;
- «хочу открыть следующего котика»;
- «жалко проигрывать, можно продолжить»;
- «надо собрать всю коллекцию».

---

## 2. Главная дизайн-цель

Главная цель — сделать игру максимально залипательной.

Для этого игра должна строиться вокруг 5 принципов:

1. **Физика всегда немного непредсказуемая.**  
   Объекты катятся, подпрыгивают, застревают, выталкивают друг друга. Игрок видит живую систему, а не сухую головоломку.

2. **Много разных объектов.**  
   Не только котики, но и молоко, корм, рыбки, клубки, игрушки, аксессуары, домики, редкие коты, магические коты.

3. **Постоянные микро-награды.**  
   Слияния, монетки, всплывающие `+2`, `+8`, комбо, новые объекты, части коллекции, ежедневные награды.

4. **Мягкое давление проигрыша.**  
   Игрок проигрывает, если поле переполнено, но перед этим есть шанс спастись бустером, рекламой или покупкой.

5. **Гибридная монетизация.**  
   Реклама для бесплатного прогресса, in-app purchases для ускорения, комфорта, отключения рекламы и косметики.

- ежедневно, чтобы забрать награду;
- периодически, чтобы открыть новые объекты коллекции.

## 4. Core gameplay

### 4.1. Основной игровой цикл

```text
Получить следующий объект
↓
Выбрать позицию сброса
↓
Сбросить объект вниз
↓
Объект падает по физике
↓
Объекты сталкиваются
↓
Одинаковые объекты сливаются
↓
Игрок получает очки, монеты, комбо
↓
Открывается следующий объект цепочки
↓
Поле постепенно заполняется
↓
Игрок пытается продержаться дольше
```

### 4.2. Вторичный цикл

```text
Играть сессию
↓
Получать монеты
↓
Покупать бустеры / скины
↓
Открывать коллекцию
↓
Возвращаться в игру
↓
Ставить новый рекорд
```

### 4.3. Долгосрочный цикл

```text
Ежедневный вход
↓
Награда / задание
↓
Открытие новых объектов
↓
Улучшение коллекции
↓
События / сезонные цепочки
↓
Возвращение в игру
```

## 5. Основная механика слияния

Игрок сбрасывает объект сверху. Объект имеет физическое тело: круг или слегка округлая форма. Если два объекта одного типа и уровня соприкасаются, они объединяются в объект следующего уровня.

Пример:

```text
Молоко + Молоко = Миска молока
Миска молока + Миска молока = Корм
Корм + Корм = Рыбка
Рыбка + Рыбка = Клубок
Клубок + Клубок = Котёнок
```

Слияние должно быть быстрым и приятным:

- лёгкий хлопок;
- дымок / облачко;
- вспышка;
- всплывающие очки;
- монетка;
- короткая вибрация на мобильном;
- новый объект слегка подпрыгивает.

## 6. Физика

### 6.1. Физический движок

Использовать Matter.js.

Физика обязательна. Она является главным источником залипательности.

Физика должна включать:

- гравитацию;
- столкновения;
- отскок;
- трение;
- вращение;
- плотность объектов;
- ограничение скорости;
- стабильное поведение в куче объектов.

### 6.2. Рекомендуемые параметры

```ts
const physicsConfig = {
  gravityY: 1,
  restitution: 0.12,
  friction: 0.85,
  frictionAir: 0.015,
  density: 0.001,
  maxVelocity: 20,
  maxAngularVelocity: 0.18,
};
```

### 6.3. Игровое поле

Поле вертикальное, как в мобильной игре.

```ts
const GAME_WIDTH = 390;
const GAME_HEIGHT = 720;
const DROP_Y = 120;
const LOSE_LINE_Y = 155;
const LOSE_DELAY_MS = 2500;
```

Физические границы:

- левая стенка;
- правая стенка;
- нижняя платформа;
- верхней стенки нет.

### 6.4. Проигрыш

Игрок проигрывает, если объекты пересекают линию проигрыша и остаются там дольше 2.5 секунд.

Важно: не проигрывать сразу, если объект просто подпрыгнул.

## 7. Главная фишка игры: разнообразие объектов

Игра не должна быть цепочкой только из котов. Чтобы она была залипательной, цепочка должна ощущаться как маленькая история превращения:

```text
Уход за котиком → игрушки → котята → породистые коты → королевские коты → магические коты
```

Игрок должен каждый раз хотеть увидеть: «а что будет дальше?».

## 10. Управление

### 10.1. На мобильных

Игрок нажимает на верхнюю часть поля, двигает палец влево-вправо и отпускает.

### 10.2. На desktop

Игрок двигает мышь по полю и нажимает ЛКМ для сброса.

### 10.3. Ограничения

- объект можно двигать только по X;
- объект нельзя вывести за стенки;
- после сброса есть задержка 400–500 мс;
- пока объект падает, уже появляется следующий preview.

## 11. Система очков

Очки начисляются за каждое слияние.

Формула:

```ts
score += objectScore[level] * comboMultiplier;
```

### 11.1. Комбо

Если слияния происходят быстро одно за другим, включается комбо.

```text
1 слияние = x1
2 слияния подряд = x2
3 слияния подряд = x3
4+ слияния подряд = x4
```

Окно комбо:

```ts
const COMBO_WINDOW_MS = 1200;
```

Комбо нужно показывать крупно и приятно:

```text
КОМБО x3!
+512
```

## 12. Валюты

### 12.1. Мягкая валюта: монеты

Монеты выдаются за:

- слияния;
- новый рекорд;
- ежедневный вход;
- задания;
- просмотр rewarded-рекламы;
- покупку in-app паков.

Монеты тратятся на:

- бустеры;
- продолжение после проигрыша;
- открытие косметики;
- ускорение прогресса;
- временные бонусы.

### 12.2. Премиум-валюта: лапки / кристаллы

Опционально можно добавить премиум-валюту.

Лучше назвать её:

```text
Лапки
```

Премиум-валюта нужна для:

- покупки редких скинов;
- покупки сезонных наборов;
- покупки больших наборов бустеров;
- покупки продолжений без рекламы;
- ускорения коллекции.

Если хочется проще, можно оставить только монеты и продавать монеты через in-app purchases.

## 13. Бустеры

Бустеры — основной инструмент монетизации и спасения игрока.

### 13.1. Кото-бомба

Удаляет выбранный объект.

Использование:

1. игрок нажимает на бустер;
2. игра переходит в режим выбора;
3. игрок нажимает на объект;
4. объект исчезает с эффектом.

Монетизация:

- можно купить за монеты;
- можно получить за rewarded video;
- можно купить набором через in-app purchase.

### 13.2. Радужный клубок

Сливает выбранный объект с ближайшим объектом такого же уровня.

Если пары нет, бустер не тратится и показывается сообщение:

```text
Нет подходящей пары
```

### 13.3. Лапка-сдвиг

Лёгкий толчок всем объектам в выбранную сторону.

Цель: помочь освободить место или подтолкнуть одинаковые объекты.

### 13.4. Кошачья печенька

Уменьшает выбранный объект на 1 уровень или превращает его в более маленький.

Использование: когда крупный объект мешает.

### 13.5. Молочный дождь

Создаёт 3 маленьких объекта сверху. Может помочь сделать серию слияний.

### 13.6. Чистая полка

Удаляет 3 самых верхних объекта. Это сильный спасательный бустер.

Лучше использовать как платный или редкий rewarded-бустер.

## 14. Проигрыш и продолжение

### 14.1. Условие проигрыша

Если объекты выше линии проигрыша остаются там 2.5 секунды, игра заканчивается.

### 14.2. Экран проигрыша

Показывать:

- грустного котика;
- текущий счёт;
- лучший счёт;
- заработанные монеты;
- кнопку «Продолжить»;
- кнопку «Рестарт»;
- кнопку «Главное меню».

### 14.3. Продолжение после проигрыша

Игрок может продолжить:

1. за rewarded video;
2. за монеты;
3. за премиум-валюту;
4. через in-app покупку набора «спасение».

После продолжения:

- удаляются 5 верхних объектов;
- линия проигрыша очищается;
- игрок получает 5 секунд защиты;
- игра продолжается с тем же счётом.

Ограничение:

- бесплатное продолжение за рекламу — 1 раз за сессию;
- платное продолжение — максимум 3 раза за сессию.

## 15. Победа

Игра бесконечная, но победа может быть локальной.

Победа срабатывает, когда игрок впервые создаёт редкий контрольный объект:

- уровень 12 — первый котёнок;
- уровень 18 — большой породистый кот;
- уровень 22 — королевский кот;
- уровень 30 — финальный кот.

### 15.1. Модалка победы

Показывать:

- заголовок: `УРОВЕНЬ ПРОЙДЕН!`;
- созданный объект;
- счёт;
- рекорд;
- награду;
- кнопку «Продолжить»;
- кнопку «Забрать x2 за рекламу»;
- кнопку «Главное меню».

### 15.2. Зачем нужна победа в бесконечной игре

Она создаёт эмоциональные пики и даёт повод показать рекламу или предложить покупку.

## 16. Коллекция

Коллекция — важный раздел удержания.

Каждый открытый объект попадает в альбом.

Для объекта показывать:

- название;
- редкость;
- сколько раз был создан;
- лучший размер комбо с ним;
- дата открытия;
- скин, если есть;
- маленькое описание.

Пример описания:

```text
Рыжий табби — любит клубки и всегда падает туда, куда не надо.
```

### 16.1. Награды коллекции

За открытие объектов:

- 5 объектов — 100 монет;
- 10 объектов — бустер;
- 15 объектов — скин;
- 20 объектов — 500 монет;
- 30 объектов — уникальный фон.

Важно: rewarded video нельзя полностью убирать, потому что это добровольная награда. Покупка `no_ads` должна отключать только принудительные межстраничные рекламные показы.

#### 4. Сезонные скины

| ID товара | Название | Содержимое |
|---|---|---|
| skin_magic_cats | Магические коты | визуальный набор объектов |
| skin_winter_cats | Зимние коты | визуальный набор объектов |
| skin_cafe_cats | Кафе котиков | визуальный набор объектов |

## 21. Реклама Яндекс Игр

### 21.1. Rewarded video

Использовать только по желанию игрока.

Места показа:

- продолжить после проигрыша;
- удвоить ежедневную награду;
- получить бесплатный бустер;
- удвоить награду после победы;
- получить монеты в магазине;
- открыть временный сундук.

Rewarded video — лучший формат для этой игры, потому что игрок сам выбирает награду.

### 21.2. Fullscreen ad

Использовать осторожно.

Места показа:

- после окончания сессии;
- при возврате в главное меню;
- после 2–3 сыгранных партий;
- не чаще, чем позволяет платформа.

Нельзя показывать fullscreen ad во время активного падения объекта или в момент, когда игрок взаимодействует с полем.

### 21.3. Sticky banner

Можно использовать на экранах меню, магазина, коллекции.

Не рекомендуется использовать внутри активного игрового поля, если баннер мешает управлению.

## 22. Экономика

### 22.1. Монеты за слияния

```ts
coins += Math.max(1, Math.floor(level / 2));
```

### 22.2. Монеты за сессию

Средняя сессия должна давать 50–200 монет.

### 22.3. Цены

Цены должны быть такими, чтобы игрок мог иногда покупать бустеры без доната, но покупка была ощутимой.

Рекомендуемые цены:

- простые бустеры: 300–500 монет;
- сильные бустеры: 800–1200 монет;
- продолжение: 700 монет;
- редкий скин: 3000–8000 монет.

### 22.4. Баланс рекламы

Игрок должен чувствовать, что реклама — это полезный бонус, а не наказание.

Пример:

- `Продолжить за рекламу` — хорошая ценность;
- `Получить x2 награду` — хорошая ценность;
- `Реклама каждые 30 секунд` — плохо.

## 34. Риски

### Риск 1: игра быстро надоедает

Решение:

- много объектов;
- коллекция;
- задания;
- события;
- комбо;
- разные наборы скинов.

### Риск 2: физика раздражает

Решение:

- мягкая гравитация;
- небольшие отскоки;
- стабильные столкновения;
- возможность использовать бустеры.

### Риск 3: монетизация слишком агрессивная

Решение:

- rewarded video только добровольно;
- fullscreen только после сессии;
- покупки не должны быть обязательными.

### Риск 4: слишком много объектов на поле

Решение:

- ограничение скорости;
- оптимизация Matter.js;
- удаление частиц;
- Canvas вместо DOM.

## 36. Требования к UX монетизации

Монетизация должна быть честной.

Нельзя:

- показывать рекламу во время активного управления;
- заставлять смотреть рекламу после каждого проигрыша;
- продавать обязательные предметы;
- делать продолжение единственным способом прогресса.

Можно:

- предлагать x2 награду;
- предлагать продолжение;
- продавать скины;
- продавать бустеры;
- продавать отключение forced-рекламы;
- продавать стартовый набор.

## 37. Итоговая формула игры

```text
Физика + милые объекты + частые слияния + коллекция + мягкая монетизация = залипательная casual-игра
```

Главный экран игры — это не магазин и не мета-прогрессия. Главный экран — физическое поле, где игрок постоянно думает:

```text
Куда сбросить следующий объект, чтобы получить цепочку слияний?
```

Все остальные разделы должны усиливать это желание:

- магазин даёт бустеры;
- коллекция даёт цель;
- ежедневные награды дают возврат;
- реклама даёт бесплатное спасение;
- покупки дают ускорение и комфорт.
